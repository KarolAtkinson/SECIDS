First-Time WebUI Guide

Open Manual
  1. Download or clone the project from GitHub: KarolAtkinson/SECIDS.
  2. Windows setup: install Python 3.10+ and Npcap, then run python -m venv .venv_test, .venv_test\Scripts\activate, and pip install -r requirements.txt.
  3. Linux/Kali setup: run python3 -m venv .venv_test, source .venv_test/bin/activate, and pip install -r requirements.txt.
  4. Start WebUI: python3 WebUI/production_server.py (Linux/Kali) or python WebUI/production_server.py (Windows), then open http://127.0.0.1:8080/login.
  5. Login steps: enter your username/password on the login page and press Login; on first sign-in, change password if prompted.
  6. Enter sudo password in Sudo Password Validation and press Enter.
  7. Use Start/Stop Scheduler and Queue Pause/Resume/Clear controls in the top toolbar when needed.
  8. Set an API token (optional) with Save Token for token-auth requests.
  9. Select a mode: Realtime or Simulation.
  10. In Simulation mode, select architecture type (school/library/restaurant), then configure attacker/defender/countermeasure profile, intensity, attackers, duration, seed, and retrain flag.
  11. Use the Models device tile to open live model selection and click Choose Model.
  12. Pick an action from the left command menu, set parameters, and click Run Action.
  13. Track execution in Execution Logs, unified operation progress, and toast event updates.
  14. Use Settings to persist default interface, duration, window, and interval preferences.
  15. Use History to review and rerun prior operations.
  16. Use Audit filters and export to validate events, roles, and statuses.
  17. Use Users management actions (admin role) to create/update/reset users when not in restricted mode.
  18. Use Model Registry to refresh/sync/register models and verify DB status.
  19. Use Packet Log taskbar tabs: Action Log and Packet Log.
  20. Use packet capture controls: Start, Pause, and Stop with live status pill.
  21. Open the Server tile to access ServerDB controls, file view, import/export, and tagged data sync.
  22. Review color-coded packet rows and fallback data behavior when live rows are unavailable.
  23. Use Logout when finished to close the active authenticated session.
