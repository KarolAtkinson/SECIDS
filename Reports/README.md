## Guidelines for Future Documentation

When creating new documentation:
1. **Always create `.md` files in the `Reports/` folder**
2. Use descriptive, UPPERCASE names with underscores: `FEATURE_NAME_REPORT.md`
3. Include date and purpose in the document header
4. Link related documentation files
5. Update this README when adding new reports

## Report Template

```markdown
# FEATURE_NAME Report

**Date:** YYYY-MM-DD  
**Author:** [Name]  
**Status:** [Active/Archived]

## Overview
Brief description of the feature/system

## Implementation Details
Technical details, code references

## Usage
How to use the feature

## Testing
Test procedures and results

## Future Enhancements
Planned improvements

---
*Last Updated: [Date]*
```
